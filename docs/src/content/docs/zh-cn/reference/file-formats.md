## `.plotx` 项目

`.plotx` 文件把整个会话保存在一个文件里：导入的数据、处理管线、分析结
果、画板排版、已保存的工作流，以及每次自动化运行的记录。因此它是自包含
的——复制到另一台机器或发给同事，打开时与你保存时完全一致，不需要记着
带任何附属文件。与使用任何分析软件一样，保留原始仪器数据仍是好习惯。

在 PlotX 1.0 之前，项目格式可能随版本变化。当较新格式的文件遇到较旧的
PlotX（或相反）时，文件会被拒绝并给出明确的"不支持的版本"提示——你的
文件绝不会被悄悄修改或迁移。遇到这种情况，更新 PlotX 后重新打开即可。

**Preferences → General → Project backup copies** 会在项目旁以隐藏文件
保留历史保存，误覆盖后可以恢复。

## `.plotxproc` 处理配方

`.plotxproc` 文件保存一条处理管线，不含任何数据——配方保存一次即可在
任何机器上应用到一整个系列的同类实验。见
[配方与模板](/zh-cn/guides/templates/)。版本处理与项目相同：不匹配的文
件会被明确拒绝，绝不会被误读。

## 工作流与运行记录文件

[自动化](/zh-cn/guides/automation/)工作流文件是一次批处理运行的 JSON
描述——导入哪些文件、应用什么、导出什么。运行后会生成一份运行记录
（manifest）文件，写明每个数据集实际发生了什么，批处理结果因此始终可
追溯。二者都是纯 JSON，均见[命令行](/zh-cn/reference/cli/)。

工作流不是配方：配方保存一条处理管线，而工作流描述一整次运行，可以把
配方作为其中一个步骤引用。
