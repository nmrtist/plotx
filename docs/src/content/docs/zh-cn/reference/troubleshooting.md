## 导入

**ABF 文件打不开。** PlotX 读取 int16 或 float32 采样的 ABF 2.x。ABF1
文件和压缩的 ABF2 数据目前不支持——请在采集软件中重新导出为未压缩的
ABF2。

**数值列被导入成了文本。** PlotX 只对无歧义的值自动定型：
`true`/`false`、十进制整数、`YYYY-MM-DD` 日期和 UTC 时间戳。区域性格式
（逗号小数、地区日期）和数字文本混排的列保持为文本，而不是被猜测。查看
**Review table import** 对话框的诊断信息，修正源文件，或导入后再转换该
列。

**Excel 表格里的单元格导入后是空的。** PlotX 读取 Excel 为每个公式缓存
的值，但从不重新计算公式。没有缓存值的公式单元格导入为空，并列在导入
诊断中——在 Excel 里打开工作簿保存一次即可刷新缓存值。

## 处理与分析

**谱峰拟合提示组分过多而拒绝运行。** 单次拟合最多处理 24 个组分。缩小
拟合范围，或删除一些峰标记。

**DOSY 图谱构建被取消了。** 图谱构建期间更改数据集的处理会取消构建，
因为图谱将不再与谱图对应。完成处理修改后重新构建即可。

**统计或时间窗统计返回错误而不是数值。** PlotX 拒绝从空窗口、非有限采
样或被排除的单元格中编造数值；消息会说明问题所在。调整窗口，或确认它
提议的排除。

## 导出

**导出预检标记了我的图。** 字号或线宽违反所选期刊预设。在画板上修正
——**Figure Typography…** 可一次设定全部坐标轴文字尺寸——或改选符合
预期尺寸的预设。

## 项目与更新

**打开项目时提示"不支持的版本"。** 该文件由项目格式不兼容的另一版本
PlotX 保存（1.0 之前的版本间可能发生）。文件完好无损——更新 PlotX 后
重试。

**我不小心覆盖了项目。** 若 **Preferences → General → Project backup
copies** 已开启，历史保存以隐藏文件保留在项目文件旁。

**收不到更新。** 失败的后台检查（例如离线时）按设计保持静默。在
**Preferences → General** 中用 **Check now** 获得明确结果，并检查更新
通道——切换到更稳定的通道不会降级，而是等待该通道的下一个版本。

## 报告问题

以上都无法解决时，请在
[GitHub issue 跟踪器](https://github.com/nmrtist/plotx/issues)报告，附上
PlotX 版本（见 **Preferences → General**）、平台，以及（可能时）能复现
问题的文件。
